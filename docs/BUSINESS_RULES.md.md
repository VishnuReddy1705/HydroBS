# HydroBS Business Rules

---

# Purpose

This document defines all business logic of HydroBS.

These rules are mandatory.

Antigravity must NEVER invent business logic.

If a requested feature contradicts these rules, these rules take precedence unless explicitly updated.

---

# Community Rules

Every Community

- Has exactly one Community Admin
- Can have many Residents
- Can have many Buildings
- Can purchase additional water
- Has its own Billing
- Has its own Reports
- Has its own Announcements
- Has its own Audit Logs

Residents from one community must never appear in another community.

# Validation Rules

All required fields must be validated.

Email must be unique.

Meter Number must be unique.

Invoice Number must be unique.

Flat Number must exist within its Community.

Billing dates cannot overlap for the same billing period.

Negative values are invalid.

---

# Error Handling Rules

Validation errors

Return descriptive messages.

Database errors

Return safe user-friendly messages.

Unexpected errors

Generate server logs.

Never expose stack traces to users.

---

# Performance Rules

Pagination for all large tables.

Lazy loading where appropriate.

Avoid duplicate database queries.

Avoid N+1 queries.

Use indexes on foreign keys.

Optimize report generation.
